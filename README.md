# Ontario Wildfire Analytics Dashboard and AFFES SharePoint Concept

This repository contains two practical portfolio projects focused on analytics, reporting, and information management.

## Project 1: Ontario Wildfire Suppression Analytics Dashboard

A 3-page Power BI dashboard built using official Ontario wildfire datasets to analyze wildfire activity trends, suppression resource rates, and fire severity patterns.

### Overview
This dashboard was created to showcase Power BI skills through a structured analytics report based on Ontario wildfire-related public data. The report combines multi-year wildfire activity reporting with 2021 listed suppression resource rates to present a decision-support style dashboard with KPI design, trend analysis, severity insights, and interactive filtering.

### Pages
1. Ontario Wildfire Overview  
2. 2021 Suppression Resource Rates  
3. Fire Severity Insights  

### Tools Used
- Power BI Desktop
- Power Query
- DAX

### Data Sources
- Annual forest fire reporting data
- Wildland fire suppression rates (2021)
- Fire disturbance spatial files for future enhancement

### Key Highlights
- Built a consistent 3-page dashboard layout
- Used multiple visuals including combo chart, gauge, matrix, donut chart, bar chart, decomposition tree, scatter chart, line chart, and waterfall chart
- Applied DAX measures for year-over-year and severity analysis
- Created a structured reporting design suitable for portfolio demonstration

### Limitations
- Suppression resource rate data currently used is limited to 2021
- Estimated Cost Proxy is a showcase metric and does not represent audited incident-level suppression spending
- Advanced geospatial reporting was planned but not fully operationalized in the current version

## Project 2: AFFES SharePoint Intranet Concept

A conceptual SharePoint intranet prototype designed to demonstrate how AFFES business areas could be organized through modern SharePoint pages with a focus on information management, reporting integration, and future workflow automation.

### Purpose
This concept was built to show how SharePoint could support:
- business-area navigation
- controlled documents
- operational notices
- request intake
- reporting access
- future Power Automate workflow integration

### Homepage Concept
The homepage was designed as a central operations and information hub for:
- Aviation Services
- Mitigation & Prevention
- IM/IT & Data Services
- Logistics & Warehousing
- Controlled Documents
- Reporting & Insights

### IM/IT & Data Services Page
A second page was created to demonstrate information management and IT support concepts, including:
- controlled documents
- metadata-style ownership and review information
- requests and forms
- reporting and dashboard preview
- future workflow automation support

### Information Management Concepts Demonstrated
- controlled document organization
- metadata and ownership
- review cycle and version status
- request and intake structure
- reporting integration
- future workflow planning through Power Automate

### Future Enhancements
- approval workflows
- document routing and notifications
- request forms connected to automation
- embedded Power BI dashboards
- review and change tracking
